# 我做了一个免费的“学术打假 AI 初筛”网站：Academic Detective

我最近做了一个公益小项目，叫 **Academic Detective**。

它的目标很简单：让任何人都可以上传一篇论文，得到一份由 AI 辅助生成的学术诚信初筛报告。

网站地址：

[https://academic-detective.pages.dev](https://academic-detective.pages.dev)

GitHub 项目：

[https://github.com/encyc/academic-detective](https://github.com/encyc/academic-detective)

这不是一个“AI 自动判定学术造假”的工具。

我不希望它变成那样。

它更像是一个低门槛的初筛助手：帮你快速扫一遍论文文本、图表描述、统计方法、结果表述和方法学细节，找出值得进一步人工复核的可疑点。

真正的学术不端认定，永远应该交给专业调查和可复核的证据链。

### 为什么想做这个项目

过去几年，学术打假的公共讨论变多了。

很多人第一次意识到，论文里的异常并不总是藏得很深。有些问题就摆在图、表、统计结果、方法描述和时间线里，只是普通读者没有足够的训练，也没有足够的时间逐项检查。

尤其是生命科学、医学、生物材料等领域，论文经常包含大量 figure、Western blot、显微图、统计图和补充材料。一个人从头到尾看完已经不容易，更别说系统地检查：

- 同一张图是否可能被复用
- 不同实验条件下的数据是否过于整齐
- 标准差、p 值、样本量是否合理
- 方法描述和结果是否互相矛盾
- 论文结论是否依赖某几个高度可疑的数据点

这些事情很适合让 AI 先做一轮“粗筛”。

不是让 AI 下结论，而是让 AI 帮人节省时间，把值得人继续看的地方标出来。

### 这个网站现在能做什么

Academic Detective 当前支持上传：

- PDF
- DOCX
- TXT

文件会先在浏览器里提取文本。也就是说，默认情况下，原始论文文件不会被上传到服务器，后端只接收提取出来的文本和文件元数据。

然后后端会调用大模型，基于内置的学术打假分析框架生成一份 Markdown 报告。

报告一般会包括：

- 论文基本信息
- 综合风险评定
- 具体可疑点
- 证据描述
- 不确定性说明
- 建议后续行动
- 免责声明

现在网站已经支持多个免费模型来源，包括 ModelScope 和 OpenCode Zen。前端可以按模型供应商分组选择具体模型。

为了防止滥用，目前每个 IP 每天有 20 次免费分析次数。页面会展示剩余次数。

### 它参考了什么分析框架

这个项目的分析框架，受到 [wooly99/geng-academic-fraud-detector](https://github.com/wooly99/geng-academic-fraud-detector) 的启发。

也向「耿同学讲故事」所代表的学术监督精神致敬🫡。

我理解的“耿同学精神”，不是简单地喊打喊杀，也不是用情绪替代证据。

它更接近四件事：

第一，有勇气提出问题。

第二，用证据说话。

第三，把复杂问题讲到普通人也能看懂。

第四，质疑论文，不搞人身攻击。

这也是这个项目想遵守的边界。

AI 可以帮我们发现疑点，但不应该替我们完成指控。一个负责任的工具，必须承认自己的不确定性。

### 它会检查哪些方向

目前内置的分析思路主要包括几类。

第一类是图片复用风险。

比如 figure caption 中声称是不同实验条件，但文本描述里出现高度相似的图像说明、重复的 control、异常一致的实验表述。由于当前版本主要处理文本，涉及真正的图片复用和拼接，还需要专业图像取证工具进一步验证。

第二类是数据异常。

比如标准差过于整齐，p 值分布可疑，不同实验组之间存在过于机械的数学关系，或者重复实验结果“完美”得不像真实实验。

第三类是统计异常。

比如小样本量却得到极其显著的结果，统计方法和数据类型不匹配，或者所有关键比较都刚好显著。

第四类是方法学矛盾。

比如方法部分写 n=5，结果表格却像 n=3；前文说使用某种实验条件，后文结果又像是另一套条件；引用文献并不能支持文中的强结论。

第五类是产出和时间线异常。

比如实验周期、投稿时间、试剂设备上市时间之间存在明显冲突。

这些都不是“定罪证据”。

但它们是值得继续追问的问题。

### 我特别想强调：这不是裁判

这个项目最重要的设计原则是：**AI 只能做初筛，不能做最终判定。**

原因很简单。

第一，AI 可能误报。

一处图片标注错误，可能是排版疏忽；一组统计结果看起来整齐，也可能来自特殊实验设计。不能因为 AI 说“可疑”，就把它当成实锤。

第二，AI 可能漏报。

真正复杂的图像操纵、跨论文图片复用、原始数据异常，很多需要专业工具、人工经验和外部数据库交叉验证。

第三，AI 看不到所有上下文。

论文正文、补充材料、原始数据、审稿记录、实验记录、作者解释，这些都可能影响判断。

所以 Academic Detective 的定位不是“自动打假神器”。

它是一个入口。

一个把检查成本降下来的入口。

### 为什么要做成 0 门槛公益项目

学术诚信不应该只属于少数专家。

很多学生、科研新手、媒体从业者、普通读者，其实都可能遇到一篇让自己觉得“不太对劲”的论文。

但他们通常没有工具，也不知道从哪里开始看。

我希望这个网站至少能提供一个起点：

上传论文，得到一份结构化初筛报告，然后根据报告里的线索，继续做人工核查。

项目目前部署在 Cloudflare Pages 上，后端使用 Cloudflare Pages Functions，模型调用优先接入免费 API。整个项目会尽量保持低成本运行。

这也是为什么我做了几个限制：

- 不需要注册
- 不收费
- 每个 IP 每天限制次数
- 默认不保存原始文件
- 明确写出免责声明

公益项目最怕两件事。

一是成本失控。

二是被滥用。

所以限制不是为了制造门槛，而是为了让它能长期运行。

### 最后

我做这个项目，并不是想鼓励大家到处“猎巫”。

恰恰相反。

学术监督越公共，越需要克制。

我们需要的是有证据、有逻辑、可复核的质疑，而不是情绪化的审判。

如果 Academic Detective 能帮一个学生更快看懂论文里的统计问题，帮一个读者发现值得追问的图表异常，或者帮一个研究者在投稿前自查潜在风险，那它就有价值。

它不会替代专业调查。

但它可以让更多人开始学会提出正确的问题。
